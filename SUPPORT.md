# Support

OmniDM is a community, self-hostable project. Support is best-effort.

## Getting help

- **Documentation** — start with [`docs/`](docs/) and the
  [README](README.md). Common topics: deployment, providers, Discord setup,
  content packs, backup/restore, migration.
- **Questions & discussion** — open a GitHub Discussion (or an issue with the
  `question` label if Discussions are not enabled).
- **Bug reports / feature requests** — use the issue templates.
- **Security issues** — **do not** open a public issue; follow
  [SECURITY.md](SECURITY.md).

## What is supported

See [docs/SUPPORT_MATRIX.md](docs/SUPPORT_MATRIX.md). Only the supported surface
receives fixes:

- Engine, web/server multiplayer, Discord, Electron (Linux), the container
  image, and content packs.

## What is not supported

- The archived Slack, Matrix, and Mattermost adapters (experimental).
- Tauri and Capacitor packaging (frozen).
- Windows/macOS code signing, notarization, and store distribution (documented
  but not provided).
- Third-party model providers you configure.

## Response expectations

This is a volunteer-maintained project. There is no SLA. Security reports are
prioritized (see [SECURITY.md](SECURITY.md)). For guaranteed/commercial support,
none is offered at this time.
